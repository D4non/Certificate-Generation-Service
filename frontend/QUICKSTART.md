# Быстрый старт

## Установка зависимостей

```bash
cd frontend
npm install
```

## Настройка переменных окружения

Создайте файл `.env` в папке `frontend/`:

```
VITE_API_URL=http://localhost:8000/api
```

Или оставьте по умолчанию (будет использован прокси из `vite.config.ts`).

## Запуск в режиме разработки

```bash
npm run dev
```

Фронтенд будет доступен по адресу: `http://localhost:3000`

## Сборка для продакшена

```bash
npm run build
```

Собранные файлы будут в папке `dist/`, их можно развернуть на любом веб-сервере.

## Структура приложения

### Страницы

- **/** - Главная страница с обзором функциональности
- **/login** - Страница авторизации
- **/upload** - Загрузка CSV/XLSX файлов с участниками
- **/templates** - Управление шаблонами сертификатов
- **/generate** - Генерация сертификатов и настройка email рассылки

### Основные компоненты

- `FileUpload` - Компонент для загрузки файлов с drag & drop
- `ParticipantsTable` - Таблица с загруженными участниками
- `Layout` - Общий layout с навигацией
- `Login` - Форма авторизации
- `ProtectedRoute` - Защита маршрутов, требующих авторизации

### API интеграция

Все API вызовы находятся в папке `src/api/`:
- `client.ts` - Базовый HTTP клиент с interceptors
- `auth.ts` - Методы авторизации
- `certificates.ts` - Методы работы с сертификатами и шаблонами

## Особенности реализации

1. **Авторизация**: Токен хранится в localStorage, автоматически добавляется к запросам
2. **Хранение данных**: Участники сохраняются в localStorage для использования между страницами
3. **Парсинг файлов**: CSV и XLSX файлы парсятся на клиенте для быстрой валидации
4. **Обработка ошибок**: Все ошибки отображаются через toast уведомления
5. **Адаптивный дизайн**: Полностью адаптивный интерфейс с поддержкой темной темы

## Требования к бэкенду API

Бэкенд должен предоставлять следующие endpoints:

- `POST /api/auth/login` - Авторизация (OAuth2 password flow)
- `GET /api/templates` - Список шаблонов
- `POST /api/templates/upload` - Загрузка шаблона
- `DELETE /api/templates/{id}` - Удаление шаблона
- `POST /api/participants/parse` - Парсинг файла участников (опционально)
- `POST /api/certificates/generate` - Генерация сертификатов
- `GET /api/certificates/{id}/download` - Скачивание сертификата
